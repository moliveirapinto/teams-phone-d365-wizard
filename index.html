Install & connect to Microsoft Teams PowerShell
Run an elevated PowerShell session. Install the module if you haven't, then sign in as a Teams admin.

3
Create the Teams resource account
Every phone number used by D365 needs its own Teams resource account, bound to the Dynamics application ID.
New-CsOnlineApplicationInstance `
  -UserPrincipalName <UPN> `
  -DisplayName "<Display name>" `
  -ApplicationId "4b8f0dce-d7d5-47a3-a27c-1764b90505e2"
i
The output includes an ObjectId — copy it into the "Object ID of resource account" field at the top so the next steps auto-fill.
!
Wait up to a few minutes for the resource account to fully propagate before running Step 4. Running too soon returns "instance not found".
4
Associate the Azure Communication Services resource
Bind the resource account to your ACS instance, then sync. This is what makes calls flow from Teams into ACS and on into D365.
Set-CsOnlineApplicationInstance `
  -Identity <Object Id> `
  -ApplicationId "4b8f0dce-d7d5-47a3-a27c-1764b90505e2" `
  -AcsResourceId "<ACS Immutable Resource ID>"

Sync-CsOnlineApplicationInstance `
  -ObjectId <Object Id> `
  -ApplicationId "4b8f0dce-d7d5-47a3-a27c-1764b90505e2" `
  -AcsResourceId "<ACS Immutable Resource ID>"

Assign a license to the resource account
In Microsoft 365 admin center, assign a Teams Phone Resource Account license (free) to the new resource account. Without it, no number can be assigned.
  1. Open the Microsoft 365 admin center → Users → Active users.
  2. Find the resource account you created in Step 3 (UPN: your UPN).
  3. Click Licenses and apps → check Microsoft Teams Phone Resource Account → Save.

Assign the phone number to the resource account
Choose your PSTN connectivity model — the cmdlets and UI flow are different for each.
Calling Plan
Operator Connect
Direct Routing
i
Calling Plan supports service numbers only here, not user numbers. Acquire one in the Teams admin center first.

Easiest: assign the service number through the Teams admin center UI.

  1. Teams admin center → Voice → Resource accounts.
  2. Select your resource account → Assign / change number.
  3. Pick the service number → Save.
Open Resource accounts ↗ 
Set-CsPhoneNumberAssignment `
  -Identity <UPN> `
  -PhoneNumber <Phone +E.164> `
  -PhoneNumberType OperatorConnect
Set-CsPhoneNumberAssignment `
  -Identity <UPN> `
  -PhoneNumber <Phone +E.164> `
  -PhoneNumberType DirectRouting

Then grant a voice routing policy so the resource account can place outbound consult/transfer calls:

Grant-CsOnlineVoiceRoutingPolicy `
  -Identity <UPN> `
  -PolicyName "<Direct Routing policy name>"
7
Sync from Teams in the D365 admin center
This is where the magic happens — D365 pulls in your Teams resource account and creates the phone number record.
  1. Open Copilot Service admin center.
  2. Site map → Customer support → Channels.
  3. On the Phone numbers row, click Manage.
  4. On the Phone numbers page, click Advanced.
  5. Open the Teams phone system tab — verify the ACS Resource ID and Dynamics App ID match yours.
  6. Click Sync from Teams — your number(s) appear as new phone-number records.
!
If sync fails, the most common causes are: missing admin consent on TeamsResourceAccount.Read.All, the resource account hasn't fully propagated, or the wrong ACS Immutable Resource ID was used in Step 4.
Open Power Platform admin ↗ Open Copilot Service admin ↗
8
Create a workstream & attach the number
Wire the new phone-number record into a voice workstream so calls actually route to your reps.
  1. Copilot Service admin center → Workstreams → + New workstream.
  2. Type = Voice, channel = Voice, set queue/routing rules.
  3. In the workstream, add a voice channel and select the Teams phone number you just synced.
  4. Configure inbound calling settings (greeting, IVR, etc.).
